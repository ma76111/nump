# دليل رفع البوت على GitHub

## ✅ التعديلات المطلوبة تمت

تم إجراء التعديلات التالية:

1. ✅ إزالة `.env` من ملف `.gitignore` - الآن يمكن رفعه على GitHub
2. ✅ تحديث `SETUP.md` مع التعليمات الكاملة
3. ✅ أزرار المهمة في القائمة السفلية (Reply Keyboard)

---

## 📤 خطوات رفع الملفات على GitHub

### الخطوة 1: إضافة الملفات المعدلة

```powershell
git add .gitignore
git add SETUP.md
git add .env
```

### الخطوة 2: عمل Commit

```powershell
git commit -m "Update: Allow .env upload and add Termux setup guide"
```

### الخطوة 3: رفع التعديلات

```powershell
git push
```

---

## 📱 بعد رفع الملفات - التشغيل على Termux

اتبع التعليمات في ملف `SETUP.md` لتشغيل البوت على هاتفك.

### ملخص سريع:

1. حمل Termux من F-Droid
2. ثبت Git و Node.js
3. اسحب المشروع من GitHub
4. شغل البوت بـ `npm start`
5. استخدم PM2 للتشغيل في الخلفية

---

## ⚠️ ملاحظات مهمة

- ✅ ملف `.env` سيتم رفعه مع الملفات
- ✅ تأكد أن المستودع **Private** (خاص) على GitHub
- ✅ التوكن موجود في `.env` ولا حاجة لتعديله على Termux
- ✅ قاعدة البيانات (*.db) لن يتم رفعها (محمية في .gitignore)

---

## 🎯 الملفات الجاهزة للرفع

الملفات التالية جاهزة وسيتم رفعها:

- ✅ `.env` (يحتوي على التوكن والإعدادات)
- ✅ `index.js` (كود البوت الرئيسي)
- ✅ `database.js` (إعداد قاعدة البيانات)
- ✅ `package.json` (المكتبات المطلوبة)
- ✅ `SETUP.md` (دليل التشغيل على Termux)
- ✅ `.gitignore` (محدث للسماح برفع .env)
- ✅ جميع الملفات الأخرى

الملفات التي **لن** يتم رفعها (محمية):

- ❌ `*.db` (قاعدة البيانات)
- ❌ `node_modules/` (المكتبات)
- ❌ `logs/` (السجلات)
